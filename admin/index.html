Privacy Policy & Legal Disclaimers

This application processes campus gate logbook scans. All data is processed using Google Gemini AI models and stored in the Google Cloud Firestore NoSQL Database.

Data Privacy Compliance
1. All photographed scans of visitor books are analyzed only for optical character recognition (OCR) and immediately discarded. Images themselves are stored securely in Firestore or deleted after processing.
2. Access is restricted to authorized personnel according to the assigned system role (Super Admin, Admin, Officer, Viewer).
3. Logs contain name, vehicle plate, time-in, destination, and other necessary entry details. No sensitive biometric or clinical information is stored.
VENLOG Admin Portal User Manual
Functionality Guide & Button Descriptions:
  • Database Tables (Sidebar): Lists all active log tables (Relations) in real-time. Use the Open button to edit records, Rename to change metadata, and Drop to delete collections.
  • Side-by-Side (Sidebar): Opens a split view displaying two sheets side-by-side. Enables comparing different logs and syncing edits to both sheets.
  • Dashboard (Sidebar): Renders Looker-style charts, graphs, and KPIs summarizing traffic, peak hours, and security metrics.
  • Settings (Sidebar): Main configuration hub for database backups, scanner limits, role white-listing, and AI credit limits.
  • Code Generator (Sidebar): Generates daily gate verification PINs required by officers before uploading photos.
  • Reload Button (Top Bar): Force-refreshes all active listeners, charts, table structures, and KPIs to fetch fresh Cloud database records.
  • Save Button (Datagrid / Compare): Pushes local JSpreadsheet table edits (new rows, text updates) securely back to Cloud Firestore.
